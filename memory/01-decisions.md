### 1. Vault 架構選擇（2026/04/17）

**議題**：選擇什麼工具和架構來管理 AI 知識庫

**選項**：
- A. Notion + GitHub
- B. Obsidian + Git（選擇）
- C. LocalStorage + 備份
- D. 其他

**決策原因**：
- Obsidian 便於本地知識管理
- Git 提供版本控制和協作能力
- 支援 Markdown，便於閱讀和編輯
- 可以輕鬆同步到 GitHub

**決策內容**：
- 使用 Obsidian 作為主要編輯工具
- 使用 Git 進行版本控制
- 將 Vault 託管在 GitHub 上
- 採用資料夾結構：identity/, context/, memory/, sop/

**執行狀態**：✅ 已完成

### 2. AI 角色定義（2026/04/17）

**議題**：定義 AI Agent 的角色和協作方式

**選項**：
- A. 被動工具（只執行指令）
- B. 主動合夥人（選擇）
- C. 學習教練（只教學）
- D. 決策者（替你做決定）

**決策原因**：
- 需要 AI 主動發現問題和風險
- 需要平衡專業指導和學習引導
- 尊重最終決策權在用戶手中
- 需要高效溝通，不浪費時間

**決策內容**：
- 角色：技術合夥人 + 學習教練
- 主動性：中等偏高（在授權範圍內主動）
- 挑戰方式：數據驅動 + 溫和提醒
- 溝通風格：直接了當、簡潔扼要

**執行狀態**：✅ 已完成
